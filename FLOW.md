# หลักการทำงานระบบชำระเงิน

เอกสารนี้อธิบาย flow การทำงานของระบบรับชำระเงินด้วย Omise ทั้ง 2 ช่องทาง

---

### ขั้นตอนการทำงาน

| ขั้น | ไฟล์ | การทำงาน |
|-----|------|---------|
| 1 | index.html | ลูกค้าเลือก "ชำระด้วยบัตรเครดิต" |
| 2 | creditcard.html | แสดงฟอร์มให้กรอกข้อมูลบัตร |
| 3 | creditcard.html | ลูกค้ากรอกข้อมูลบัตร |
| 4-6 | Omise.js | สร้าง Token ที่ browser (ไม่ส่งข้อมูลบัตรเข้า server) |
| 7 | creditcard.html | ได้รับ Token จาก Omise |
| 8 | checkout.php | ส่ง Token ไปประมวลผลที่ server |
| 9-10 | checkout.php | เรียก Omise API เพื่อ charge จริง |
| 11 | checkout.php | แสดงผลสำเร็จหรือล้มเหลว |

### จุดสำคัญ

- **ความปลอดภัย**: ข้อมูลบัตรไม่เคยเข้า server ของเรา
- **PCI Compliance**: ลดข้อกำหนดการรับรองความปลอดภัย
- **Token**: ใช้ครั้งเดียวแล้วทิ้ง มีอายุสั้น

### ขั้นตอนการทำงาน

| ขั้น | ไฟล์ | การทำงาน |
|-----|------|---------|
| 1 | index.html | ลูกค้าเลือก "ชำระด้วยพร้อมเพย์" |
| 2-4 | promptpay.php | สร้าง Charge กับ Omise ขอ QR Code |
| 5 | promptpay.php | แสดง QR Code ให้ลูกค้า |
| 6-7 | แอพธนาคาร | ลูกค้าสแกน QR และชำระเงิน |
| 8 | check-status.php | ลูกค้ากดตรวจสอบสถานะ |
| 9-10 | check-status.php | ตรวจสอบกับ Omise API |
| 11 | check-status.php | แสดงผล pending / successful / failed |

### สถานะการชำระเงิน

| สถานะ | ความหมาย | การแสดงผล |
|-------|---------|----------|
| pending | รอการชำระเงิน | แสดง QR Code ต่อ + นับถอยหลัง |
| successful | ชำระเงินสำเร็จ | แสดงหน้าสำเร็จ |
| failed | ชำระเงินล้มเหลว | แสดงข้อผิดพลาด + ให้ลองใหม่ |

## ไฟล์ที่เกี่ยวข้อง

```
form-example/
├── index.html          # หน้าเลือกวิธีชำระเงิน
├── creditcard.html     # ฟอร์มบัตรเครดิต
├── promptpay.php       # สร้าง QR Code
├── checkout.php        # ประมวลผลบัตรเครดิต
├── check-status.php    # ตรวจสอบสถานะ PromptPay
└── config.php          # ตั้งค่า Keys
```

## ความแตกต่างระหว่าง 2 ระบบ

| หัวข้อ | บัตรเครดิต | PromptPay |
|-------|-----------|-----------|
| การกรอกข้อมูล | ที่หน้าเว็บ | ผ่านแอพธนาคาร |
| ความเร็ว | ทันที | รอลูกค้าสแกน |
| ความปลอดภัย | ใช้ Tokenization | ไม่เก็บข้อมูลบัตร |
| ค่าธรรมเนียม | ~3% | ถูกกว่า |
| การยืนยัน | ทันที | ต้องตรวจสอบสถานะ |
